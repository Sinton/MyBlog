站点级信息集中在 `src/theme.config.ts`。这一步像给报纸换刊名：不动版式，先把站点身份、栏目顺序、版次信息和外链换成自己的。

## 报头信息

`siteTitle` 用于 `<title>`、默认描述、报头可访问文本和顶部可见的大刊名。默认情况下，只改这一项就能换掉报头标题。

`siteSubtitle` 显示在顶部黑条，`issue`、`since`、`location` 和 `frequency` 显示在报头两侧。

```ts
export const themeConfig = {
  siteTitle: "铅字旧报",
  siteSubtitle: "Astro Newspaper Theme",
  issue: "第一版 第一号",
  since: "2026",
  location: "Theme Desk",
  frequency: "周刊",
  backToTopLabel: "回到刊首",
  enableSmoothScroll: true,
};
```

如果你只想把“我的晚报”换成自己的刊名，不需要改组件：

```ts
export const themeConfig = {
  siteTitle: "阿通周报",
};
```

主题会自动把 `siteTitle` 拆成单字渲染，并保留旧报纸题头的紧凑排版。

如果你想逐字微调报头，可以使用 `mastheadGlyphs`。它会覆盖默认的 `siteTitle` 拆字渲染：

```ts
export const themeConfig = {
  siteTitle: "阿通周报",
  mastheadGlyphs: [
    { text: "阿", tone: "strong" },
    { text: "通" },
    { text: "周", tone: "wide" },
    { text: "报", tone: "strong" },
  ],
};
```

`tone` 是可选项，支持 `default`、`narrow`、`wide` 和 `strong`。普通站点不需要配置它；只有想调整某个字的宽窄或墨印感时再使用。

## 导航

`navItems` 决定主导航顺序。推荐保留“头版 -> 案例 -> 专栏 -> 说明书 -> 关于”的节奏：先看样张，再看文章，再查说明书，最后了解主题。

```ts
navItems: [
  { label: "头版", href: "/" },
  { label: "案例", href: "/projects/" },
  { label: "专栏", href: "/articles/" },
  { label: "说明书", href: "/docs/" },
  { label: "关于", href: "/about/" },
],
```

新增页面时，先在 `src/pages` 下创建路由，再把入口加入 `navItems`。比如要加 `/notes/`，可以创建 `src/pages/notes/index.astro`，再增加：

```ts
{ label: "札记", href: "/notes/" }
```

## 社交链接

`socialLinks` 的地址会被报头右上角的 SVG 图标读取。默认组件保留 GitHub 和 X 两个写死图标，只把链接地址交给配置文件。

如果你要增加 RSS、邮箱或其他图标，需要在 `Masthead.astro` 里补一个对应的 SVG 块，再从 `themeConfig.socialLinks` 里读取地址。

## 页面描述

`description` 会进入默认 meta description，也会被首页等页面作为站点介绍使用。把它写成一句能说明站点定位的话，不要只写“个人博客”。

## 滚动交互

`enableSmoothScroll` 控制是否启用主题内置的平滑滚动。默认值是 `true`，会挂载 `SmoothScroll`，文章锚点、目次跳转和回到顶部都会走同一套滚动逻辑。

如果你更喜欢浏览器原生滚动，或者站点里已经接入了自己的滚动方案，可以把它关掉：

```ts
export const themeConfig = {
  enableSmoothScroll: false,
};
```

关闭以后，`BackToTop` 仍然保留，会使用浏览器原生的 `window.scrollTo` 回到顶部。

`backToTopLabel` 控制右下角回顶按钮的文案。默认是：

```ts
export const themeConfig = {
  backToTopLabel: "回到刊首",
};
```
