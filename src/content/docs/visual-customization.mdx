建议先替换内容，再改视觉。这样你能判断哪些样式是真的需要改，哪些只是样稿不适合你的站点。

## 先改配置

站点名、副标题、导航、版次和社交链接都在 `src/theme.config.ts`。如果只是换站点身份，不需要动组件。

报头可见的四个大字在 `src/components/Masthead.astro`。主题默认按四字中文报头设计，如果你要换成两个字、五个字或英文刊名，需要同步调整 `masthead__title` 和 `masthead__glyph` 相关样式。

## 再改颜色

主要颜色集中在 `src/styles/global.css` 的 `:root`：

```css
:root {
  --paper: #efe2c2;
  --paper-light: #f8efdc;
  --ink: #17130f;
  --stamp-red: #9e2f24;
  --dispatch-blue: #2f536b;
}
```

不要一次替换太多颜色。旧报纸风格依赖纸色、墨色、分隔线和少量印章色，颜色太满会让页面不像报纸。

## 保留版面节奏

主题的识别度主要来自这些细节：

- 报头和版次信息。
- 双线分隔和细线网格。
- 专栏列表里的日期、栏目和摘要。
- 文章页的索引词和目次。
- 案例页的档案栏和功能分栏。

如果要大改布局，优先保留这些节奏。它们比单独的背景纹理更能决定主题气质。

## 检查移动端

改完视觉后至少检查首页、文章页和案例页的移动端。重点看报头是否过高、导航是否换行自然、长标题是否压住正文。
